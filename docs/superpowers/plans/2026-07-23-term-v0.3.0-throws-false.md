# term v0.3.0 `{ throws: false }` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an opt-in `{ throws: false }` structured-result variant to `waitFor`/`waitForQuiet` so direct lib importers can probe-then-branch without `try/catch`.

**Architecture:** The shared `poll()` helper becomes a pure function returning an internal `WaitForResult` union (never throws). The public `waitFor`/`waitForQuiet` wrappers own the throw/return policy at the API boundary â€” `throws:true` (default) throws `TermTimeoutError` (unchanged v0.2.0 behavior); `throws:false` returns a `WaitForResult` discriminated union on `ok`. Function overloads keep the default path typed as `CaptureResult` so existing `r.text` callers are unaffected. The pi tool surface (`extensions/term.ts`, `lib/dispatch.ts`) is unchanged â€” the tool's `isError`-on-timeout already serves the LLM agent.

**Tech Stack:** TypeScript (raw `.ts`, tsx runtime), node:test, typebox/pi-ai schema, tmux 3.7+, pnpm. No build step.

**Spec:** `docs/superpowers/specs/2026-07-23-term-v0.3.0-throws-false-design.md`
